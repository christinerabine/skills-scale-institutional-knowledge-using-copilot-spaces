# OctoAcme Role-Specific Onboarding Checklist

## Purpose
Provide structured onboarding guidance for each role to ensure new team members can quickly become productive contributors to OctoAcme projects.

---

## General Onboarding (All Roles)

### Week 1: Orientation
- [ ] Complete HR/IT onboarding and system access setup
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) to understand all team roles
- [ ] Review [Process Ownership Matrix](octoacme-process-ownership-matrix.md) for your role
- [ ] Meet your manager and discuss initial goals
- [ ] Join team Slack channels and mailing lists
- [ ] Attend team standups as an observer
- [ ] Shadow 1-2 team members in your role

### Week 2-4: Integration
- [ ] Review current project documentation and backlogs
- [ ] Attend sprint planning, demos, and retrospectives
- [ ] Schedule 1:1 meetings with key stakeholders
- [ ] Begin contributing to team activities with mentorship
- [ ] Complete security and compliance training

---

## Project Manager Onboarding

### Technical Setup
- [ ] Access to project management tools (GitHub Projects, Jira, etc.)
- [ ] Access to documentation repositories and wikis
- [ ] Calendar permissions for scheduling meetings
- [ ] Access to risk register and status report templates

### Knowledge Areas
- [ ] Review [Project Initiation Guide](octoacme-project-initiation.md)
- [ ] Review [Project Planning](octoacme-project-planning.md)
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md)
- [ ] Review [Risk Management & Communication](octoacme-risks-and-communication.md)
- [ ] Review [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Review [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md)

### Role-Specific Tasks
- [ ] Shadow an experienced PM through a full sprint cycle
- [ ] Co-facilitate a standup or planning meeting
- [ ] Review 3-5 completed project one-pagers
- [ ] Practice using the risk register template
- [ ] Create your first status update with mentor review
- [ ] Attend weekly PM sync meetings

### First 90 Days
- [ ] Lead your first project kickoff meeting
- [ ] Own a small-to-medium project from planning to delivery
- [ ] Facilitate a retrospective session
- [ ] Build relationships with Product Managers and stakeholders
- [ ] Identify one process improvement opportunity

## Product Manager Onboarding

### Technical Setup
- [ ] Access to analytics and metrics dashboards
- [ ] Access to customer feedback tools and CRM
- [ ] Access to product roadmap and backlog tools
- [ ] Access to user research repositories

### Knowledge Areas
- [ ] Review product vision, strategy, and roadmap
- [ ] Understand current customer segments and personas
- [ ] Review key success metrics and OKRs
- [ ] Study competitive landscape and market positioning
- [ ] Review recent customer research and feedback

### Role-Specific Tasks
- [ ] Shadow product planning and backlog refinement sessions
- [ ] Review acceptance criteria and user story writing guidelines
- [ ] Meet with Sales, Support, and Customer Success teams
- [ ] Review recent release notes and feature launches
- [ ] Participate in customer interviews or demos
- [ ] Attend weekly PM/PdM sync meetings

### First 90 Days
- [ ] Lead backlog grooming for one feature area
- [ ] Define success metrics for a new feature
- [ ] Conduct user research or usability testing session
- [ ] Present roadmap update to stakeholders
- [ ] Collaborate with PM on project one-pager
- [ ] Make prioritization decisions for your product area

## Developer Onboarding

### Technical Setup
- [ ] Development environment setup (IDEs, SDKs, tools)
- [ ] Code repository access and branching strategy review
- [ ] CI/CD pipeline access and understanding
- [ ] Local build and test environment working
- [ ] Debugger and profiling tools configured

### Knowledge Areas
- [ ] Review architecture documentation and design patterns
- [ ] Understand coding standards and style guides
- [ ] Review PR workflow and code review guidelines
- [ ] Study test coverage requirements and testing strategy
- [ ] Review security best practices and vulnerability scanning

### Role-Specific Tasks
- [ ] Clone repositories and run local builds successfully
- [ ] Complete "good first issue" or small bug fix
- [ ] Submit your first PR and go through code review
- [ ] Review 5-10 existing PRs to understand team standards
- [ ] Pair program with experienced team member
- [ ] Review Definition of Done checklist

### First 90 Days
- [ ] Implement a small-to-medium feature end-to-end
- [ ] Participate actively in code reviews
- [ ] Write comprehensive tests for your code
- [ ] Fix a production bug or customer-reported issue
- [ ] Contribute to technical design discussions
- [ ] Mentor or pair with another new developer

## Tracking Your Progress

### Self-Assessment
- Review this checklist weekly with your manager or mentor
- Mark items complete as you gain proficiency
- Ask for help or clarification on any unclear items
- Document questions or areas needing more support

### Manager Support
- Schedule regular check-ins to review progress
- Provide introductions to key stakeholders
- Assign appropriate mentors or buddies
- Adjust onboarding pace based on individual needs

### 30/60/90 Day Reviews
- **30 Days**: Complete general onboarding and begin role-specific tasks
- **60 Days**: Contributing independently with some guidance
- **90 Days**: Fully productive team member, ready for increased responsibility
